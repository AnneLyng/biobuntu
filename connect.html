Preparation

In all of the following *Terminal* codes you should replace bio737 with your own KU-id. Shout out loud if you run into unclear statements, typos or real problems …

Biostat computing servers

The computing servers with ssh access are

  • doob (virtual)
  • rasch (virtual)
  • gauss (48 cores, anarchy)
  • borel (48 cores, anarchy)
  • bayes (64 cores, cue system)
  • fisher (64 cores, cue system)

To connect with one of these machines you need to either be connected via cable at CSS or open a vpn connection (see instructions below). Then, you can login to doob via: 

ssh -X bio737@doob

Hard to remember?

Add aliases to the file ~/.bashrc 

alias gauss='ssh -X bio737@gauss'
alias borel='ssh -X bio737@borel'
alias fisher='ssh -X bio737@fisher'
alias bayes='ssh -X bio737@bayes'
alias doob='ssh -X bio737@doob'

wicd: Eduroam template

Step 1

Create a file as sudo: file:///etc/wicd/encryption/templates/eduroam Insert the following and save the file: 

name = EDUROAM
author = ElitestFX
version = 1
require identity *Identity password *Password
-----
ctrl_interface=/var/run/wpa_supplicant
network={
        ssid="$_ESSID"
        scan_ssid=$_SCAN
        key_mgmt=WPA-EAP
        eap=PEAP
        phase2="auth=MSCHAPV2"
        identity="$_IDENTITY"
        password="$_PASSWORD"
}

Step 2

Add a line with the word eduroam to the file (edit as sudo) file:///etc/wicd/encryption/templates/active

Step 3

launch wicd-gtk, select eduroam under encryption and specify properties for eduroam:

identity = bio737@ku.dk password your KU-password

Open vpn (wireless connection outside campus)

Install openconnect: 

sudo apt-get install openconnect --assume-yes

To open the vpn connection, You will need the sudo password of your ubuntu machine and the password for your KU-id: 

sudo openconnect --juniper -u bio737 vpn.sund.ku.dk

At the second prompt: 

realm [sund.ku.dk|Consultant|KU access]:

type sund.ku.dk followed by the ENTER/RETURN key.

Hard to remember?

Typing v in a terminal is easier to remember and less typing effort than sudo openconnect --juniper -u bio737 vpn.sund.ku.dk. To use this, simply, add the following line to the file ~/.bashrc: 

alias v='sudo openconnect --juniper -u bio737 vpn.sund.ku.dk'

Tired of typing realm and password?

Could not find a very nice solution yet, but …

  • Install xdotool
sudo apt-get install xdotool  --assume-yes
  • Create a shell script in your local bin, e.g., as ~/bin/vpn, where you also substitute with your own password:
xterm &
sleep 0.5
xdotool type "sudo openconnect --juniper -u bio737 vpn.sund.ku.dk"
xdotool key Return
sleep 0.5
xdotool type "sund.ku.dk"
xdotool key Return
sleep 0.5
xdotool type "xxxxxx"
xdotool key Return
  • Make the script executable but remove read/write access for other people:
chmod 700 ~/bin/vpn
  • Execute shell script in terminal via
sudo ~/bin/vpn
  • Create alias
alias v='sudo ~/bin/vpn' # shell script

Connect to server without typing the password

Open vpn (if it does not exist yet)

Follow instructions above.

Create folder .ssh on server (if needed)

ssh -X bio737@doob
mkdir ~/.ssh

Generate key

ssh-keygen -t rsa -C "bio737@sund.ku.dk"

Export key

cat .ssh/id_rsa.pub | ssh bio737@doob 'cat >> .ssh/authorized_keys'
